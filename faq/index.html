How to share your Pixie library with viewers using a chatbot command

The Pixies Config panel can export a PNG image of your full sprite library. You can upload that image somewhere and create a chatbot command like !pixies so viewers know which sprites are available.

1
Export the library image. In Casterlabs, open the ChatIncluded Pixies Config tab. At the top of the panel, click Save Library as PNG. The file is saved to your Downloads folder as chatincluded-pixies.png.
2
Upload it to Discord to get a permanent link. Open any Discord server or DM and send the chatincluded-pixies.png file as a message. Once it appears in chat, right-click the image and choose Copy Link. This gives you a direct URL ending in .png that anyone can open in a browser.
3
Create the chatbot command. In whichever chatbot you use (Nightbot, StreamElements, Fossabot, etc.), create a command — for example !pixies — and set the response to something like:

Choose your pixie with !pixie [name] — see all options here: https://cdn.discordapp.com/attachments/.../chatincluded-pixies.png

Paste the Discord link you copied in step 2 in place of the example URL.
4
Re-export when your library changes. If you add or remove custom pixies later, repeat steps 1–2 and update the link in your chatbot command.

Troubleshooting

Common issues

After updating, my settings are missing or the widget behaves strangely

When a new version of ChatIncluded is installed, Casterlabs may hold on to cached data from the old widget. The fix is to remove your existing ChatIncluded widgets and re-add them fresh:

  1. In Casterlabs, go to Widgets & Alerts.
  2. Delete any existing ChatIncluded widgets (translator, Pixies overlay, Pixies Config).
  3. Re-add them via + → Other and reconfigure your settings.

Your custom Pixies are stored separately and will not be lost when you remove and re-add the widgets.

The installer gets stuck or shows an error about a locked file

Close Casterlabs before running the installer. The plugin file is loaded by Casterlabs while it is open, which prevents the installer from replacing it. Quit Casterlabs completely, run the installer, then reopen Casterlabs.
